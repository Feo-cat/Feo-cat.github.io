Gin 入门学习笔记：先把一个 API 跑起来

发表于2026-05-17|更新于2026-05-17|技术笔记
|浏览量:

最近在补 Go 后端相关的东西，Gin 算是绕不开的一站。它不像 Spring 那样一上来就有很多概念，也不像直接写 net/http 那么“原生态”，整体感觉是：先让 API 跑起来，再慢慢把路由、参数、JSON 和中间件这些东西串上。

这篇就当作我的第一轮 Gin 学习记录，不追求把所有能力一次讲完，先把我觉得最核心、最容易马上用起来的部分整理清楚。

我对 Gin 的第一印象

Gin 是基于 Go 标准库 net/http 封装出来的 Web 框架。它主要解决的是这些问题：

  • 路由写起来更直观
  • 请求参数获取更方便
  • JSON 返回更顺手
  • 中间件可以统一处理日志、鉴权、异常恢复等逻辑
  • 项目稍微变大之后，可以用路由分组把接口组织起来

如果只写一个简单接口，net/http 也能完成；但当接口多起来以后，Gin 的结构会更清晰。这也是我决定先学 Gin 的原因：它足够轻，又离真实项目足够近。

参数：Path、Query 和 Body 分开看

刚开始学 Web 框架时，我最容易混的是参数来源。现在我会把它们分成三类：

  • Path 参数：放在 URL 路径里，比如 /users/1001
  • Query 参数：放在 ? 后面，比如 /search?keyword=gin
  • Body 参数：放在请求体里，常见于 POST/PUT 的 JSON

小结

Gin 给我的感觉是：它没有把 Go 的 Web 开发包得太厚，但又补上了日常写 API 时最需要的便利性。

这一轮我重点记住了四件事：

  • gin.Default() 可以快速启动一个带基础中间件的服务
  • 路由负责把 HTTP 方法和 URL 映射到处理函数
  • 参数要分清 Path、Query 和 Body
  • 中间件适合处理日志、鉴权、恢复、限流这类公共逻辑

等这些基础写顺手之后，再去接数据库、鉴权和部署，心里会稳很多。
